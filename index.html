Edit Content

Elements Panel allows us to edit the content of any DOM node.

We can check the overflow behavior of the nav bar item (li element) on the left by editing its .text content in the Elements panel.

Edit Tags

Allows us to edit tags.

Change the p tag of the nav item to a span to make it semantically accurate and see if that effects its layout or style.

Edit/Add Attributes

Allows us to edit/add attributes on an element.

Add a data attribute data-badge and set its value as 10 on the li element of the nav bar item.

Move Nodes

Allows us to move DOM nodes around.

Let's say we want to check how it likes if the icon is on the right side of the text for the nav bar item. We drag and drop the .text element above the .icon.

Delete Nodes

Allows us to delete DOM nodes.

We can check if the nav menu borders maintain their curved state even when we have less items by simply selecting nav bar item node and hitting DEL key.

Edit HTML

Allows us to edit any node as HTML by right clicking on a DOM element and choosing "Edit as HTML".

We can check if the nav menu borders maintain their curved state even when we have less items by simply selecting nav bar item node and hitting DEL key.

Tip: Access Inspected Element in Console

We can access element instance of the latest inspected element by typing $0 in the console.

$0 changes automatically to the selected element in the Elements Panel.

Older inspected elements can also be accessed like $1, $2, ...

Tip: Navigate through the Hierarchy Using Breadcrumbs

To go to the parent of the inspected element it is very helpful to use breadcrumbs especially when

  • there are many children to the parent (which makes us scroll more to reach the parent)
  • hierarchy is too deep

Elements Panel: DOM Breakpoints

Hover on me to see a custom tooltip.

Break on subtree modifications

This allows us to pause the execution on any subtree modifications such as node adds/removals.

This tooltip here is added to document body when hovered on the element and removed from it immediately after we move away from the hovered element. In this situation, to inspect the tooltip or to know which code adds or removes the tooltip, we can set a subtree modifications breakpoint on the body.

More breaking options

In addition to subtree modifications we also have

  • attribute modifications - which allows us to break on attribute changes such as a class got added to the clnode removal, etc
  • node removal - which breaks when the current node on which the breakpoint is going to be removed from the DOM

We can see the list of all these breakpoints in "DOM Breakpoints" panel.

Elements Panel: Edit Styles

Edit Styles

Styles Panel allows us to edit styles to test different combinations until we are satisfied with output.

Let's say we want to tweak the border-radius of the button, we can find it under .btn class in the Styles Panel.

Add Styles

We also can add new styles to the element.

Click on the negative space beside .btn class styles. This will add an empty text area under it to allow us to add a new style property for that class. Let's add the following.

border: 0;

Add Pseudo States

Allows us to add/edit pseudo states (such as :hover, :visited) for an element.

While the button is selected in the Elements Panel, click on + (New Style Rule) button and edit button.btn class name that has been added, to .btn:hover and hit Panel key. Add the following under that hover state style rule.

background: var(--default-dark-color);

Add class

Allows us to add a new class to the selected element.

Add a new class by clicking .cls button under Styles Panel and typing in the new class name. For our button you can type primary as the class name and hit Enter to ad that class.

Fix the Accessibility of the Text

Dev tools gives contrast ratio for the text color we have chosen based on its background which indicates whether the color is legible enough. And also suggests us best range of the color to choose from for better visibility.

We see that the text on the button after setting the class .primary is not legible. Dev tools indicates the same. We can expand to more details of the contrast ratio.

We can set to a more visible color

  • either by click on the refresh icons beside the contrast ratio for AA and AAA, or
  • choosing a color which falls within the ranges of AA and AAA that are indicated on the color panel

Tip: Inspector Stylesheet

Any added styles (ex. hover state that we have added for the .btn class) can be found in the temporary inspector stylesheet that the browser creates.

Tip: Toggle Pseudo States

We can toggle different pseudo states on an element using the :hov button.

Tip: Changes Panel

The Changes Panel helps us see all the changes we have done to existing CSS in one place as a diff.

Tip: Computed Styles

The Computed Panel gives us the following capabilities.

  • Box model visualization
  • Final computed value of each CSS property
  • A hierarchical view of each property on the selected (or inspected) element. In other words, it gives us all the cascading overrides that have be applied on that specific property.

Tip: Shortcut Properties

When using shorthand CSS properties such as background, the Styles panel shows on expansion all the lower level properties with their assigned values based on the shorthand property value.

Edit a Layout

Specifically for flex layout, chrome dev tools offers a quick way to edit it.

We can change the direction of the card content to column instead of row by using the GUI flexbox tool that dev tools provide. Toggle the column direction button from the GUI flexbox popover seen below.

The popover adopts to the direction change and shows the corresponding property icons. Change the align-items to flex-end by toggling the corresponding button.

Toggle a Layout

Dev tools allow us to visualize a layout on the viewport by creating a dashed border around it on toggling the related switch from the Elements Panel.

We can change the direction of the card content to column instead of row by using the GUI flexbox tool that dev tools provide. Toggle the column direction button from the GUI flexbox popover seen below.

Layout Panel

All the layouts (flex and grid) that are available on the DOM can be viewed in the Layout Panel.

This panel also allows us to

  • toggle these layouts for visualization using the checkbox (same as toggling it from the Elements Panel)
  • add more details for the layout items (grid area names, track sizes, etc)
  • change the color of the layout visualization for better accessibility

Toggle the grid layout of the .card to see the layout and check all the options under Grid section to see various details of the laid out items. Also change the color of the layout to improve visibility.

Elements Panel: Animations

The Animations panel allows us to

  • inspect animations by slowing them down, replaying them, or viewing their source code.
  • modify animations by changing their timing, delay, duration, or keyframe offsets.

Tweak the marked handles on one of the eye ball by moving them on the timeline and see what happens.

Tip: GUI CSS Property Editors

Chrome dev tools provides GUI editors for some of the properties such as color, easing, flexbox layout, etc which can be quite beneficial while tweaking those properties.
