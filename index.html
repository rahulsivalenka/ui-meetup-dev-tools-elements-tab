Elements Tab: Edit DOM

Edit Content

Elements Tab allows us to edit the content of any DOM node.

We can check the overflow behavior of the nav bar item (li element) on the left by editing its .text content in the Elements tab.

Edit Tags

Allows us to edit tags.

Change the p tag of the nav item to a span to make it semantically accurate and see if that effects its layout or style.

Edit/Add Attributes

Allows us to edit/add attributes on an element.

Add a data attribute data-badge and set its value as 10 on the li element of the nav bar item.

Move Nodes

Allows us to move DOM nodes around.

Let's say we want to check how it likes if the icon is on the right side of the text for the nav bar item. We drag and drop the .text element above the .icon.

Delete Nodes

Allows us to delete DOM nodes.

We can check if the nav menu borders maintain their curved state even when we have less items by simply selecting nav bar item node and hitting DEL key.

Edit HTML

Allows us to edit any node as HTML by right clicking on a DOM element and choosing "Edit as HTML".

We can check if the nav menu borders maintain their curved state even when we have less items by simply selecting nav bar item node and hitting DEL key.

Tip: Access Inspected Element in Console

We can access element instance of the latest inspected element by typing $0 in the console.

$0 changes automatically to the selected element in the Elements Tab.

Older inspected elements can also be accessed like $1, $2, ...

Tip: Navigate through the Hierarchy Using Breadcrumbs

To go to the parent of the inspected element it is very helpful to use breadcrumbs especially when

  • there are many children to the parent (which makes us scroll more to reach the parent)
  • hierarchy is too deep

Elements Tab: DOM Breakpoints

Hover on me to see a custom tooltip.

Break on subtree modifications

This allows us to pause the execution on any subtree modifications such as node adds/removals.

This tooltip here is added to document body when hovered on the element and removed from it immediately after we move away from the hovered element. In this situation, to inspect the tooltip or to know which code adds or removes the tooltip, we can set a subtree modifications breakpoint on the body.

More breaking options

In addition to subtree modifications we also have

  • attribute modifications - which allows us to break on attribute changes such as a class got added to the clnode removal, etc
  • node removal - which breaks when the current node on which the breakpoint is going to be removed from the DOM

We can see the list of all these breakpoints in "DOM Breakpoints" tab.

Elements Tab: Edit Styles

Edit Styles

Styles Tab allows us to edit styles to test different combinations until we are satisfied with output.

Let's say we want to tweak the border-radius of the button, we can find it under .btn class in the Styles Tab.

Add Styles

We also can add new styles to the element.

Click on the negative space beside .btn class styles. This will add an empty text area under it to allow us to add a new style property for that class. Let's add the following.

border: 0;

Add Pseudo States

Allows us to add/edit pseudo states (such as :hover, :visited) for an element.

While the button is selected in the Elements Tab, click on + (New Style Rule) button and edit button.btn class name that has been added, to .btn:hover and hit Tab key. Add the following under that hover state style rule.

background: var(--default-dark-color);

Add class

Allows us to add/edit pseudo states (such as :hover, :visited) for an element.

While the button is selected in the Elements Tab, click on + (New Style Rule) button and edit button.btn class name that has been added, to .btn:hover and hit Tab key. Add the following under that hover state style rule.

background: var(--default-dark-color);

Tip: Inspector Stylesheet

Any added styles (ex. hover state that we have added for the .btn class) can be found in the temporary inspector stylesheet that the browser creates.

Tip: Toggle Pseudo States

We can toggle different pseudo states on an element using the :hov button.

Tip: Changes Tab

The Changes Tab helps us see all the changes we have done to existing CSS in one place as a diff.
